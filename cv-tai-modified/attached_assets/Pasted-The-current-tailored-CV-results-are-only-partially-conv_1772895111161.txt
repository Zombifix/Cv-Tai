The current tailored CV results are only partially convincing. The system selects plausible bullets, but it still does not produce a truly tailored CV. I need a stronger generation pipeline.

Main issues observed:
1. Target role is often "Unknown" or only partially parsed.
2. Detected keywords section is empty, which means the job parsing / explainability layer is weak.
3. The final CV feels like a list of locally matched bullets, not a coherent targeted application narrative.
4. The generated skills section is wrong and sometimes includes unrelated skills like Node.js / React / TypeScript for a Product Designer profile.
5. Safe mode and ATS mode are not differentiated enough.

I want the pipeline to be refactored into 2-level matching:

Step 1: Parse the job description into a structured object:
- title
- company
- seniority
- responsibilities
- required skills
- preferred skills
- domain / industry
- keywords

Step 2: Score experiences first
For each experience, compute a relevance score based on:
- role similarity
- industry similarity
- skill overlap
- semantic similarity with responsibilities
- recency

Step 3: Score bullets only inside the top-ranked experiences
Do not rank all bullets globally first.
Select the top experiences, then top bullets within them.

Step 4: Build a coherent CV composition
Generate:
- targeted title
- short summary aligned with the job
- selected experiences in best order
- selected bullets per experience
- relevant skills only if they are present in selected experiences or validated profile skills

Step 5: Fix the Optimization Report
It must show:
- parsed job title
- parsed company
- detected keywords
- selected experiences with why they were selected
- selected bullets with reason
- missing skills / gaps
- confidence level

Step 6: Fix skills generation
Never inject generic or unrelated technical skills.
Only show skills that:
- exist in the user library
- are linked to selected experiences
- are relevant to the parsed job

Step 7: Differentiate the two modes more clearly
Safe Mode:
- preserve original wording as much as possible
- reorder and select only

ATS Mode:
- allow light rewriting
- normalize phrasing
- embed relevant keywords from the job naturally
- no hallucination

Step 8: Add debug visibility
I want logs or a developer view for:
- parsed job object
- experience scores
- bullet scores
- final composition plan
- rejected bullets

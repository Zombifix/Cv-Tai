You are Replit Agent. Build an MVP web app called “CV Tailor” for tech jobs.

Goal
- Single user MVP (no auth required for now), but structure code so adding multi-user + auth later is easy.
- User maintains a “Super-CV library” (a long, detailed master resume) with many experiences + many bullet points.
- User inputs a job posting via URL (primary) and optional paste text (fallback).
- The app:
  1) extracts job content from URL (server-side scrape)
  2) semantically matches job requirements to the user’s Super-CV bullets
  3) selects the best experiences + bullets (transparent selection)
  4) produces a tailored CV text output + a structured report:
     - “Selected bullets (kept)” vs “Candidate bullets (considered)”
     - keywords covered / missing (ATS-ish)
     - short tips for improvement and optional “variant angle” suggestion (e.g., AI angle), but NEVER invent experience.

Non-negotiables / Safety
- DO NOT invent experiences, companies, achievements, tools, or metrics.
- All rewritten bullets must be traceable to source bullets selected from the library.
- If information is missing, state it as a gap and suggest what to add, without fabricating.
- Always show “sources” for each rewritten bullet (which original bullet(s) it came from).

MVP scope decisions
- Target: tech roles only (engineering/product/design/DS/PM etc.).
- Input job posting: URL first; also provide a fallback textarea for copy/paste if scrape fails.
- Company news/market research: NOT in MVP. (Add later as Phase 2)
- Output: plain text CV (copy/paste). No export (PDF/DOCX) in MVP.

Tech stack (optimize for Replit)
- Next.js (App Router) + TypeScript
- Tailwind for styling
- Supabase Postgres for persistence (even for single user), using pgvector for embeddings.
- OpenAI API:
  - embeddings model for semantic search
  - chat model for rewriting & reporting
- Use server actions / route handlers for API logic. Keep all secrets in env vars.

Embedding / Retrieval strategy
- When user creates/edits a bullet, generate embedding and store it.
- When a job post is processed, extract structured signals:
  - role title, seniority signals, responsibilities, requirements, tools/keywords
  - also produce a “job_summary” text used for embedding query
- Generate embedding for job_summary and retrieve top-N bullets by cosine similarity.
- Selection rules:
  - Group bullets by experience, compute score per experience.
  - Pick top 4 experiences (configurable) with highest aggregated relevance + priority boosts.
  - For each selected experience, pick top 4–6 bullets with best relevance, respecting bullet priority.
  - Keep a “considered bullets list” (top 12 globally) for transparency.

Rewrite modes
- Mode “safe”: minimal edits, improve verbs/clarity, keep meaning extremely close. No keyword stuffing.
- Mode “ats”: controlled rewrite, incorporate relevant keywords ONLY if they are already true per source bullets.
- In both modes: output must include mapping from rewritten bullet -> source bullet ids.
